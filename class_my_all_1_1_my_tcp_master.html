Detailed Description

My master class for TCP connection.

This is the master class of MyTcpClient & MyTcpServer. This class has many common members of these child classes. Be careful not to operate this alone.

Previous check before receiving & waits for the time-out interval. :

This is the preparing function before receiving process.
The function checks the receiving status and wait connections when the time-out function is set. If the return value(n_read_) is <0, the error is occured. Error codes are as belows,.

  • -3 : Connection error before the receiving process. (without time-out setting)
  • -4 : Connection error before the receiving process. (with time-out setting)
  • -5 : Time-out error before the receiving process.
  • -6 : There is no readable data in the socket before the receiving process.

If immediately after these happen, you can get error messages with ErrMsg().

Parameters
[in]useSock__: A socket number. (file descriptor) If this is default(=-1), useSock_ is taken on.
Returns
the receivable code(=1) or error code (<0).

Previous check before sending & waits for the time-out interval. :

This is the preparing function before sending process.
The function checks the sending status and wait when the time-out function is set. Note that this function should be used with Send(). If the return value(n_write_) is <0, the error is occured. Error codes are as belows,.

  • -3 : Connection error before the sending process. (without time-out setting)
  • -4 : Connection error before the sending process. (with time-out setting)
  • -5 : Time-out error before the sending process.
  • -6 : Disconnection detection before the sending process. (This is checked CheckConnect().)
  • -7 : There is no sendable file descriptor before the sending process.

If immediately after these happen, you can get error messages with ErrMsg(). Please note finely when you wanna re-write this, because this code is designed for speed.

Parameters
[in]useSock__: A socket number you wanna receive. (file descriptor) If this is default(=-1), useSock_ is taken on.
[in]pre_check__: Connection check before sending. default is true.
Returns
The sendable code(=1) or error code (<0).

Receives messages (char) :

This is the function for receiving with TCP protocol.
If the return value(n_read_) is >= char array size, you should repeat the receiving processes. If the return value(n_read_) is <0, the error is occured. Error codes are as belows,.

  • 0 : Non received data (possibility of non connections, already)
  • -1 : Receiving error.
  • -2 : Non initialized error.
  • -3 : Connection error before the receiving process. (without time-out setting)
  • -4 : Connection error before the receiving process. (with time-out setting)
  • -5 : Time-out error before the receiving process.
  • -6 : There is no readable data in the socket before the receiving process.
  • -7 : Disconnection detection before the receiving process.

If immediately after these happen, you can get error messages with ErrMsg().

Parameters
[in]data__: A char array (char pointer) to store received data. Please set an char array after the declaration like "char data[N]". This array should be initialized using memset by yourself.
[in]sizeof_data__: The maximum byte size to store received data. you should set "sizeof(data__)". (data__ is the first argument.)
[in]useSock__: A socket number. (file descriptor) if this is default(=-1), useSock_ is taken on.
Returns
The byte size of received data or error code (<0).
